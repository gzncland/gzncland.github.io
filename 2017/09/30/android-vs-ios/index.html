Android与iOS在开发中的对比

发表于 2017-09-30

总览

本文基于 Android 7.x 及 iOS8，从应用生命周期、界面差异、后台机制上对比了 Android 与 iOS 在开发上的部分区别。

生命周期

主要分析Application和UIApplication的生命周期、Activity和UIViewController的生命周期。Android独有的Service和ContentProvider的生命周期不在此列出。

Application vs UIApplication

Android

onCreate/onTerminate

此方法在应用程序创建/终止时调用。

onTrimMemory/onLowMemory

  • onTrimMemory：此方法在系统出现内存不足时调用。会有一个参数来获取系统目前的内存状态。
  • onLowMemory：当所有后台进程被终止后系统内存仍然不足时调用。

onConfigurationChanged

当系统语言被切换、横竖屏转换、换手机卡等配置更变时会被调用。

iOS

willFinishLaunchingWithOptions/didFinishLaunchingWithOptions

应用启动后将被调用。对应onCreate

applicationWillResignActive/applicationDidEnterBackground

  • applicationWillResignActive：应用即将切换到非活跃状态时发出通知。比如用户按下了Home键回到桌面。

  • applicationDidEnterBackground：应用已经进入后台时调用。比如用户按下了Home键回到桌面，或者切入了别的程序。

applicationWillEnterForeground/applicationDidBecomeActive

  • applicationWillEnterForeground：应用从后台进入前台时调用。
  • applicationDidBecomeActive：应用从非活跃状态切换到前台时调用。应用启动时也会调用此方法。

applicationDidReceiveMemoryWarning

系统内存不足，会调用此方法。对应onTrimMemory

applicationWillTerminate

应用将要被终止时调用。比如用户双击Home键后清除了程序。对应onTerminate。

Activity vs UIViewController

Activity

onCreate/onDestory

Activity创建/销毁时调用。

onStart/onRestart/onResume

Activity进入前台时调用。

  • onStart/onRestart：当Activity从不可见状态进入可见状态时。
  • onResume：用户已经可以与应用交互。

onPause/onStop

Activity切入后台时调用。

  • onPause：Activity失去焦点，比如低电量提示弹窗之类的Dialog弹出，但是Activity还处于可见状态。自己弹出Dialog时不会进入onPause。
  • onStop：Activity已经于不可见状态。在内存不足的情况下这个方法可能不会被执行到

onTrimMemory/onLowMemory

见上方Applation说明

UIViewController

initWithCoder/initWithFrame/initWithNibName

分别使用StoryBoard/代码/Xib创建该Controller时调用。

loadView/viewDidLoad

对应onCreate

  • loadView：view为nil时，会调用此方法创建view。
  • viewDidLoad：view创建完成，初始化界面。

viewWillAppear/viewDidAppear

对应onStart、onResume。在两个方法中间会调用viewWillLayoutSubviews及viewDidLayoutSubviews。

  • viewWillAppear：在视图即将展现的时候会调用。比如：想要从本级返回上级界面时，上级界面的viewWillAppear会先于调用本页面的viewDidDisappear调用，因为系统感知到上级view将要被展示。
  • viewDidAppear：在视图已经展现完的时候调用。

viewWillLayoutSubviews/viewDidLayoutSubviews

子视图发生变化时会调用。例如屏幕旋转时。

viewWillDisappear/viewDidDisappear

对应onPause、onStop

  • viewWillDisappear：视图将要消失时，例如想要返回上级页面时，开始滑动此页面但还没有完全回到上级页面时。
  • viewDidDisappear：视图已经消失不可见。比如被新的页面覆盖或者从页面栈中退出。

viewDidUnload/didReceiveMemoryWarning

对应onTrimMemory``onLowMemory。系统内存不足，会调用此方法。

init/dealloc

UIViewController初始化、销毁时调用。

界面

界面跳转

Android

  • Intent：Android应用通过Intent跳转页面，通过向系统提交Intent，会跳转对应的页面（如果有）。适用于应用内跳转或者跨应用调起界面。

iOS

  • UINavigationController：在应用内跳转界面使用UINavigationController进行跳转。

Toolbar vs UINavigationBar

Toolbar

可扩展、灵活的工具栏。不同Activity中可以使用不同Toolbar。

UINavigationBar

基于UINavigationController内的导航栏。多个UIViewController在一个UINavigationController内共用一个UINavigationBar。

RecycleView vs UITableView

RecycleView

Android用于代替ListView的控件。通过ViewHolder复用提升性能，并且加入LayoutManager作出横向滑动或者网格型列表等排列形式。每行高度可以通过使用warp_content让系统自动计算高度。

UITableView

通过UITableViewCell复用提升性能，需要自定义高度。

后台

后台任务

Android

在Android系统中，应用程序切换至后台时，不会影响其任务的运行。即使Activity结束，也可以通过Service挂起前台服务进行持续性任务。

iOS

在iOS系统中，普通应用在用户按下Home键，返回到桌面后，应用的applicationDidEnterBackground委托方法会被调用。此时iOS会给应用一段时间来保存内容。当超过限定的时间后，应用将被冻结。

推送

安卓

没错，确实是安卓而不是Android。在安卓中，想要获得服务端的推送，需与服务器实现长连接，然后根据服务端的推送协议弹出Notifycation，或其他界面刷新操作。

iOS

iOS通过APNs（Apple Push Notification service)进行推送，开发者需要在Apple后台生成推送证书，并进行绑定。之后上架AppStore的程序在安装时会把设备的token分享给App，App通过token推送对应消息到设备上。推送由APNs在设备上保持长连接，开发者无需（也不能）在后台自行实现长推送服务。
